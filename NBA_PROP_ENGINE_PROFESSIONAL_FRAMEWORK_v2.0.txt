SECTION 1 - PLAYER INJURY STATUS AND AVAILABILITY

Purpose:
Establish whether the player is active, restricted, questionable, or fully out.
No prop evaluation has structural validity without verified same-day status.

Fields collected:
- player_status
- injury_description
- restriction_severity
- last_status_update_timestamp
- minutes_restriction_flag
- same_day_status_resolution

Sources:
- official.nba.com/nba-injury-report-2025-26-season
- covers.com/sport/basketball/nba/injuries
- rotowire.com/basketball/injury-report.php

Staleness rule:
- same-day sensitive
- maximum 4 hours stale before tipoff

Hard-fail conditions:
- HF-01 if same-day status cannot be verified
- HF-03 if same-day sources conflict and conflict is unresolved
- HF-07 if official designation is OUT or INACTIVE
- HF-08 if official NBA injury report is inaccessible

HF-02 REWRITE

HF-02: MINUTES RESTRICTION FLAG IS TRUE AND SAME-DAY CONFIRMATION DOES NOT
CLARIFY THE SPECIFIC CAP

Condition:
minutes_restriction_flag = TRUE AND the source (official injury report or
same-day team statement) confirms a restriction exists but does not state the
specific cap or expected minutes range.

Rationale:
A restriction without a stated cap cannot be modeled because the range of
possible outcomes spans from 0 to 40 minutes. A projection built on that
uncertainty is structurally invalid.

Exception:
If the restriction is confirmed AND a specific cap or minutes estimate is
stated, HF-02 does NOT trigger. Produce a constrained projection at the stated
cap with explicit MODEL ESTIMATE label.

PART 4 - FORMULAS AND MODEL PARAMETERS
================================================================================

4.1 DE-VIGGING

Classification:
MATHEMATICALLY CORRECT - not a model parameter

Formula:
For + odds:
Impl = 100 / (odds + 100)

For - odds:
Impl = |odds| / (|odds| + 100)

Fair_Over = Impl_Over / (Impl_Over + Impl_Under)

Validation rule:
Fair_Over + Fair_Under must equal 1.000 +/- 0.001

4.2 MOMENTUM DECAY

Classification:
[MODEL PARAMETER]

Formula:
P_true = (L3 x 0.60) + (L5 x 0.30) + (L8 x 0.10)

Disclosure:
Weights are engine design values, not empirically validated decay constants.
Treat as tunable.

4.3 USG REDISTRIBUTION

Classification:
[MODEL PARAMETER]

Formula:
USG_adj = USG_baseline x (1 + 0.15 x (USG_absent / 100))

Disclosure:
The 0.15 multiplier is a design-set value, not a published empirical constant.

4.4 ROLE FRAGILITY

Classification:
[MODEL PARAMETER]

Formula:
role_fragility = (minute_volatility_score x 0.5) +
((1.0 - last_n_role_consistency) x 0.5)

Volatile trigger:
>= 0.55

Disclosure:
Weighting and cutoff are design parameters.

4.5 BLOWOUT THRESHOLDS

Classification:
[MODEL PARAMETER]

Thresholds:
- Spread >= 15 = HIGH (10% to 15% reduction on favored team)
- Spread 10 to 14.5 = MEDIUM (5% to 10% reduction)
- Spread < 10 = LOW
- B2B + spread >= 10 = ELEVATED (+5%)
- underdog adjustment = 5% to 8% in ELIMINATED or LOCKED_OUT_RISK blowout-loss
  scenarios

Disclosure:
Thresholds and reduction bands are design parameters, not empirically validated
league constants.

4.6 HOME / AWAY COEFFICIENTS

Classification:
[MODEL PARAMETER]

Values:
- HOME = 1.00
- AWAY = 0.97
- AWAY B2B cross-country = 0.94

Disclosure:
These are engine-design coefficients and must be treated as tunable.

PART 7 - CONFIDENCE SCORING AND OUTPUT FORMAT
================================================================================

7.1 CONFIDENCE CLASSES

HIGH
- all core fields present
- all live fields fresh
- no unresolved source conflicts
- no major proxy substitutions

MEDIUM
- one or two secondary fields missing
- or one proxy source required
- no hard-fail event

LOW
- important contextual field missing
- or major proxy dependence required
- or section conflict resolved only with limited confidence
- no hard-fail event

UNDETERMINED
- used when an optional but structurally meaningful layer, such as historical
  referee tendency data, is unavailable and cannot be responsibly estimated

- player
- prop_type
- line
- side
- model_prob
- fair_prob
- edge
- confidence_class
- hard_fail_flags
- notes

- All design-set coefficients or thresholds must retain [MODEL PARAMETER].
